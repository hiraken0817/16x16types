# 16²（256 TYPES）— 実装仕様書

Web性格診断サービス「16²」の本実装のための引き継ぎドキュメント。
プロトタイプ検証済みの確定事項と、未確定の設計課題を区別して記載する。

---

## 1. プロダクト概要

- 表層タイプ（世間に見えている/自認している4文字タイプ）×深層タイプ（内に宿るタイプ）＝ **16×16＝256サブタイプ** の性格診断
- 各結果は幻獣として表現：二つ名「**{深層の幻獣}宿せし{表層の役職名}**」（例：ドライアド宿せし司会者）、コード表記「**ESFJ - INFP**」（表層 - 深層）
- ビジネスモデル：診断無料＋タイプ別解説書（有料）販売
- 用語は「**表層**」「**深層**」で統一（「表面」「器」「認知」は不採用）
- 純血型（表層＝深層）の特別演出は**不要**（決定済み）

## 2. 法的制約（最重要・変更不可）

1. 「MBTI」「Myers-Briggs」「16Personalities」の語を**コード・UI・マーケ文言・メタタグ・広告のいずれにも使用しない**。MBTIは日本で商標登録済み（日本MBTI協会、16/35/41類）で、権利者は能動的に警告を行っている
2. 16Personalitiesの解説文の**参照・翻案を禁止**。全コンテンツ完全独自執筆（NERIS Analytics Ltdが全IPを留保、商用利用禁止）
3. 16Personalities訳語（建築家・提唱者・分析家など）は全廃済み。**data/types.json の役職名が最終確定版**
4. グループ名（分析家・外交官・番人・探検家）は**未差し替え・要対応**（types.jsonにnote記載）
5. 断定的運命論の禁止：「絶対に避けるべき職業」等の断定はNG。「消耗しやすい働き方」「この環境では幻獣が枯れる」等の環境・条件ベース表現に置き換える
6. 4文字コード（INTJ等）自体の使用はリスク低（商標登録の形跡なし）
7. 販売開始前に弁護士・弁理士確認を行う

## 3. 確定データ（data/）

- `data/types.json` — 表層役職名16・幻獣16・グループ・色・二つ名フォーマット。**単一ソースとしてここだけを参照すること**
- `data/questions.json` — 全36問（表層12問＋深層8方向×3問）と採点仕様。プロトタイプ実装から機械抽出済み
- 幻獣画像：`assets/beasts/{CODE}.webp` 16体（透過・最大820px・計約1.2MB）

## 4. デザイン仕様（プロトタイプ検証済み）

### 二層構造「表＝光、深層＝闇」
- **診断フロー（トップ〜設問）**：ライト基調。和紙アイボリー `#f6f3ea` × 墨紺 `#221f33`。白カード `#fffdf7`、実線ボーダー
- **結果画面・図鑑のみ**：ダークファンタジー。背景 `#08080d`、深層グループ色のオーラ（脈動）、幻獣浮遊、粒子、段階フェード
- フォント：見出し=Shippori Mincho B1、本文=Zen Kaku Gothic New
- グループ色はライト用・ダーク用の2系統（types.json参照）。文字用途はさらに明るいaccent（例：SJ青の文字は #6fade8）

### 視認性規範（過去に問題が起きた領域・必須）
- 全文字色はWCAG AA（4.5:1）以上をビルド時に機械チェック
- **html/bodyの背景に依存しない**：全コンテンツを `.page` ラッパーdivで包み、背景はそこに持たせる（プレビュー環境でbody背景が剥がされる事例があった）
- `<meta name="color-scheme">` を明示
- 納品前にスクリーンショット確認（通常環境＋html/body背景を白に強制した敵対環境の2パターン）

### 画面フロー（7画面）
1. トップ `/` — ロゴ・「扉をひらく」（ロゴ回転吸い込み演出）
2. 表層入力 `/start` — 16タイプグリッド（主動線）／簡易12問への分岐
3. 簡易診断 `/quiz/surface` — カード式・縦4ボタン（A/どちらかといえばA/どちらかといえばB/B）
4. 中間画面 — 表層確定の橋渡し演出
5. 深層診断 `/quiz/deep` — 表層の4文字に対応する4方向×3問=12問
6. 結果 `/result/{表層}-{深層}` — 256静的ページ
7. 図鑑 `/beasts` — 幻獣16体グリッド
- **設問画面では軸名・傾向タグ（本物/隠れ・排除/統合）を絶対に表示しない**（回答バイアス防止）

## 5. 採点仕様

### 表層（第1部・確定）
- 4軸×3問。重み：核2.0／裏1.5／軽1.0
- 4択の強度：両端1.0／中間0.5。選んだ側の文字に 重み×強度 を加算
- 各軸で合計が大きい側の文字を採用（同点はA側=先頭文字）

### 深層（第2部・⚠️仮ロジック、本設計が必要）
- 現状のプロトタイプ実装：各方向でB側（隠れ/統合）比率≥50%なら文字反転、未満なら維持
- **問題点**：E/I・J/Pの「本物/隠れ」は反転解釈が自然だが、S/N・T/Fの「排除型/統合型」は劣等機能の成熟度であり、反転とは概念が異なる
- **本設計で決めること**：8方向スコア→深層16タイプへの写像ルール、閾値、同点処理、ゲージ表示値との整合、テストケース作成
- これが**最優先の未解決課題**。実装前にテスト駆動で仕様を固めること

## 9. 画像パイプライン（新幻獣差し替え時の手順）

1. 元素材：PNG埋め込み型SVG（`href="data:image/png;base64,..."`）
2. base64抽出 → PIL読み込み → 透過確認（透過なしは背景除去が必要：純白 R=G=B のみ除去、色味のある白は残す）
3. `getbbox()`で透明部トリム → `thumbnail(820,820)` → WebP quality=84
4. ビルドスクリプト化してリポジトリに含めること
